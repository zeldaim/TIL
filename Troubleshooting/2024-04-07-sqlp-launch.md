\# 📚 \[TIL] Windows 환경 세팅 트러블슈팅: PowerShell 보안 및 환경 변수(Path) 설정



\## 1. 개요

\* \*\*날짜:\*\* 2026-04-07

\* \*\*배경:\*\* SQLP 공부를 위해 D 드라이브에 오라클(Oracle XE) 및 파이썬 가상환경(venv)을 구축하던 중 발생한 실행 오류 해결 과정.



\---



\## 2. 문제 상황 1: PowerShell 스크립트 실행 권한 오류

\### ❌ 에러 메시지

> `이 시스템에서 스크립트를 실행할 수 없으므로... Activate.ps1 파일을 로드할 수 없습니다.`



\### 🔍 원인 분석

윈도우의 기본 보안 정책인 \*\*Execution Policy\*\*가 `Restricted`(제한됨)로 설정되어 있어, 직접 만든 파이썬 가상환경 활성화 스크립트(`.ps1`) 실행을 차단함.



\### ✅ 해결 방법

터미널에서 실행 권한을 `RemoteSigned`(로컬 스크립트는 허용)로 변경함.

```powershell

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

```



\---



\## 3. 문제 상황 2: `sqlplus` 명령어 미인식 (CommandNotFound)

\### ❌ 에러 메시지

> `sqlplus : 'sqlplus' 용어가 cmdlet, 함수, 스크립트 파일 또는 실행할 수 있는 프로그램 이름으로 인식되지 않습니다.`



\### 🔍 원인 분석

오라클을 기본 경로(C:)가 아닌 \*\*D 드라이브\*\*에 설치함에 따라, 윈도우 시스템이 실행 파일(`sqlplus.exe`)의 위치를 파악하지 못함. \*\*시스템 환경 변수(Path)\*\*에 해당 경로가 등록되지 않음.



\### 🛠️ 해결 과정 (트러블슈팅 실습)

1\.  \*\*파일 탐색:\*\* 터미널 명령어로 실제 설치 경로를 추적함.

&#x20;   ```powershell

&#x20;   Get-ChildItem -Path D:\\ -Filter sqlplus.exe -Recurse

&#x20;   # 실제 경로 확인: D:\\Dev\\Oracle\\dbhomeXE\\bin

&#x20;   ```

2\.  \*\*환경 변수 수동 등록:\*\* 윈도우 \[시스템 환경 변수 편집] → \[Path] 편집 → 신규 경로 추가.

&#x20;   \* 추가 경로: `D:\\Dev\\Oracle\\dbhomeXE\\bin`



\---



\## 4. 학습 포인트 \& 회고

\* \*\*인프라 격리:\*\* 파이썬 가상환경(`venv`)을 통해 프로젝트별 독립된 환경을 구축하는 법을 익힘.

\* \*\*환경 변수의 이해:\*\* OS가 명령어를 실행할 때 찾아보는 '지도'와 같은 `Path`의 원리를 체득함.

\* \*\*결과:\*\* 이제 어느 위치에서든 `sqlplus`를 호출하여 오라클 데이터베이스에 접속할 수 있는 환경을 완비함.
