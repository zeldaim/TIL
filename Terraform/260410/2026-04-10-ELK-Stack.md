\# Terraform으로 ELK Stack 인프라 코드화



\## 환경

\- Rocky Linux 9

\- Terraform 최신버전

\- Docker Provider (kreuzwerker/docker \~> 3.0)



\## 설치

sudo dnf install -y yum-utils

sudo yum-config-manager --add-repo https://rpm.releases.hashicorp.com/RHEL/hashicorp.repo

sudo dnf install -y terraform



\## 파일 구조

\~/terraform-soc/

├── main.tf   ← Provider 설정

└── elk.tf    ← ELK Stack 리소스 정의



\## main.tf

terraform {

&#x20; required\_providers {

&#x20;   docker = {

&#x20;     source  = "kreuzwerker/docker"

&#x20;     version = "\~> 3.0"

&#x20;   }

&#x20; }

}

provider "docker" {}



\## elk.tf

resource "docker\_image" "elasticsearch" {

&#x20; name = "docker.elastic.co/elasticsearch/elasticsearch:8.12.0"

&#x20; keep\_locally = true

}



resource "docker\_container" "elasticsearch" {

&#x20; image = docker\_image.elasticsearch.image\_id

&#x20; name  = "elasticsearch-tf"

&#x20; ports {

&#x20;   internal = 9200

&#x20;   external = 9202

&#x20; }

&#x20; env = \[

&#x20;   "discovery.type=single-node",

&#x20;   "xpack.security.enabled=false",

&#x20;   "ES\_JAVA\_OPTS=-Xms512m -Xmx512m"

&#x20; ]

}



\## 명령어

terraform init    # 초기화

terraform plan    # 변경사항 미리보기

terraform apply   # 실제 적용

terraform destroy # 인프라 삭제



\## 결과

\- elasticsearch-tf 컨테이너 포트 9202 ✅

\- kibana-tf 컨테이너 포트 5602 ✅



\## 배운 것

\- Terraform으로 Docker 컨테이너를 코드로 관리

\- keep\_locally = true → 이미지 재사용 (다운로드 생략)

\- depends\_on → 컨테이너 실행 순서 보장

\- terraform plan으로 적용 전 변경사항 확인 가능

\- 포트 충돌 방지 위해 기존 ELK와 다른 포트 사용 (9202, 5602)



\## AWS 이식 시

provider "docker" {} → provider "aws" {}

docker\_container → aws\_instance

로 변경하면 동일한 구조로 클라우드 배포 가능
