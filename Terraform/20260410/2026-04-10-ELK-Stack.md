# Terraform으로 ELK Stack 인프라 코드화

## 환경

- Rocky Linux 9
- Terraform 최신버전
- Docker Provider (kreuzwerker/docker ~> 3.0)

## 설치

```bash
sudo dnf install -y yum-utils
sudo yum-config-manager --add-repo https://rpm.releases.hashicorp.com/RHEL/hashicorp.repo
sudo dnf install -y terraform
```

## 파일 구조

```
~/terraform-soc/
├── main.tf   ← Provider 설정
└── elk.tf    ← ELK Stack 리소스 정의
```

## main.tf

```hcl
terraform {
  required_providers {
    docker = {
      source  = "kreuzwerker/docker"
      version = "~> 3.0"
    }
  }
}

provider "docker" {}
```

## elk.tf

```hcl
resource "docker_image" "elasticsearch" {
  name         = "docker.elastic.co/elasticsearch/elasticsearch:8.12.0"
  keep_locally = true
}

resource "docker_container" "elasticsearch" {
  image = docker_image.elasticsearch.image_id
  name  = "elasticsearch-tf"

  ports {
    internal = 9200
    external = 9202
  }

  env = [
    "discovery.type=single-node",
    "xpack.security.enabled=false",
    "ES_JAVA_OPTS=-Xms512m -Xmx512m"
  ]
}

resource "docker_image" "kibana" {
  name         = "docker.elastic.co/kibana/kibana:8.12.0"
  keep_locally = true
}

resource "docker_container" "kibana" {
  image = docker_image.kibana.image_id
  name  = "kibana-tf"

  ports {
    internal = 5601
    external = 5602
  }

  env = [
    "ELASTICSEARCH_HOSTS=http://elasticsearch-tf:9202"
  ]

  depends_on = [docker_container.elasticsearch]
}
```

## 명령어

```bash
terraform init    # 초기화
terraform plan    # 변경사항 미리보기
terraform apply   # 실제 적용
terraform destroy # 인프라 삭제
```

## 결과

- `elasticsearch-tf` 컨테이너 포트 9202 ✅
- `kibana-tf` 컨테이너 포트 5602 ✅

## 배운 것

- Terraform으로 Docker 컨테이너를 코드로 관리
- `keep_locally = true` → 이미지 재사용 (다운로드 생략)
- `depends_on` → 컨테이너 실행 순서 보장
- `terraform plan`으로 적용 전 변경사항 확인 가능
- 포트 충돌 방지 위해 기존 ELK와 다른 포트 사용 (9202, 5602)

## AWS 이식 시

```hcl
# Docker provider → AWS provider로 전환
provider "docker" {}  →  provider "aws" { region = "ap-northeast-2" }

# 컨테이너 → EC2 인스턴스로 전환
docker_container  →  aws_instance + user_data(docker-compose 스크립트)
docker_image      →  AMI ID로 대체
```

동일한 모듈 구조 유지하면서 provider만 교체하는 방식으로 클라우드 배포 가능
