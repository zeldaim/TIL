## 환경 이슈 및 해결

### iptables PORTSCAN 룰 재설정
VM 재부팅으로 iptables 초기화됨 → 재설정 필요

```bash
sudo iptables -A INPUT -p tcp --tcp-flags SYN,ACK,FIN,RST RST -m limit --limit 1/s -j LOG --log-prefix "PORTSCAN: "
sudo iptables -A INPUT -m state --state NEW -p tcp --dport 0:1024 -j LOG --log-prefix "PORTSCAN: "
```

**교훈**: iptables는 재부팅 시 초기화됨. 영구 저장 필요:
```bash
sudo iptables-save > /etc/iptables/rules.v4
```

### Wazuh Agent 비활성화 → 활성화
```bash
sudo systemctl start wazuh-agent
sudo systemctl enable wazuh-agent
```

### 방화벽 HTTP 포트 오픈
```bash
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --reload
```

## 시나리오 2 — T1190 Exploit Public-Facing Application (SQLi)

### 취약한 웹앱 셋업 (Rocky Linux)
```bash
# Apache + PHP 설치
sudo dnf install -y php php-mysqli php-fpm

# MariaDB 설정
sudo mysql -u root
CREATE DATABASE testdb;
CREATE TABLE testdb.users (id INT, username VARCHAR(50));
INSERT INTO testdb.users VALUES (1, 'admin');
CREATE USER 'webuser'@'localhost' IDENTIFIED BY 'webpass';
GRANT ALL ON testdb.* TO 'webuser'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

취약한 PHP 페이지 (`/var/www/html/login.php`):
```php
<?php
$conn = new mysqli("localhost", "webuser", "webpass", "testdb");
$user = $_GET["user"];
$query = "SELECT * FROM users WHERE username = '$user'";
$result = $conn->query($query);
echo $query;
?>
```

### 공격 (Kali)
```bash
curl "http://192.168.190.128/login.php?user=admin%20union%20select%201"
```

### 탐지 결과
```json
{
  "rule": {
    "id": "100005",
    "level": 10,
    "description": "SQL Injection attempt detected",
    "mitre": { "id": ["T1190"] }
  },
  "data": {
    "srcip": "192.168.190.130",
    "url": "/login.php?user=admin%20union%20select%201"
  }
}
```

**탐지 경로**: Kali → Apache → /var/log/httpd/access_log → Wazuh Agent → Wazuh Manager

### 핵심 교훈
- Wazuh 룰 100005가 `union`, `select` 키워드 탐지
- `OR '1'='1'` 페이로드는 룰에 없어서 탐지 안 됨 → 룰 개선 필요 (detection-gaps)
